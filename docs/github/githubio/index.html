    github.io(github pages) 사용법

    github.io는 github에서 무료로 제공하는 정적 호스팅 서비스이다.

    아래 설명은 개발자가 아닌 분들을 대상으로 하였다.

    github에 접속

    https://github.com 으로 접속

    github 회원가입 / 로그인

    회원가입을 위해 sign up 버튼을 누른다.

    회원 가입 관련 화면은 여기까지만.

    이후 설명은 회원 가입 완료 후 아래 그림과 같이 로그인(sign in) 버튼을 눌러 로그인에 성공한 이후부터다.

    레포지토리(저장소) 생성

    레포지토리는 나의 html, js, css 파일 등을 저장하는 공간이다. 인터넷에 연결된 서버에 저장되어 전 세계 어디에서도 이 정보에 접근할 수 있다.

    대시보드에서 [new] 버튼을 눌러 새로운 레포지토리 생성.

    다음과 같이 설정한다.

    • 레포지토리 이름: html5
    • public 선택
    • Add a README file 생성에 체크

    그리고 맨 아래 [Create repository] 버튼을 클릭한다.

    레포지토리는 위와 같은 모습으로 생성되어야 한다.

    이 웹브라우저는 이 문서의 끝까지 닫지 않고 놔둔다.

    레포지토리를 내 PC로 가져오기

    github에 만든, 서버에 있는 레포지토리를 내 PC로 가져온다.

    복사해 오기 때문에 clone 이라고 한다.

    Github 사용법이 조금 복잡하니 조금이라도 편하게 쓰라고 만들어 놓은 github desktop을 이용해 레포지토리를 내 PC로 복제해 온다.

    github desktop 설치

    그런데 github desktop이 설치되어 있지 않기 때문에 다운로드 받아 사용해야 한다.

    github desktop을 다운로드 받아 설치한다.

    github desktop을 실행한다.

    github가 로그인되어 있는 웹브라우저를 제외한 다른 브라우저는 모두 닫는다.

    위 버튼을 눌러 github.com에 로그인(sign in) 한다.

    그럼 아래와 같이 github 웹브라우저에서 로그인하겠다고 나온다.

    [continue] 버튼을 눌러 로그인 처리한다.

    그럼 아래처럼 github desktop을 열거냐고 물어본다.

    github desktop을 열겠다는 버튼을 눌러준다.

    그러면 이렇게 웹의 정보를 이용해 자동으로 로그인이 된다.

    파란색 [Finish] 버튼을 눌러준다.

    github desktop을 이용해 레포지토리 복제

    github desktop이 실행되면 왼쪽에서 레포지토리 목록을 선택하고 아래에서 복제 버튼을 눌러준다.

    그럼 어디에 복제해 놓겠냐고 물어보는데, 위치를 정해주고 [Clone] 버튼을 눌러준다.

    이제 github에 있던 레포지토리가 내 PC로 복사되어 내려온다.

    잘 복제되었는지 확인

    내 경우 d:\test\html5에 복제되라고 설정했었다.

    d:\test\html5 폴더에 가보면 이렇게 새로운 html5라는 폴더가 생성되고 그 안에 뭔가 내용이 들어 있는 걸 볼 수 있다.

    서버의 내용이 잘 복제되었다.

    내가 만든 파일을 서버에 업로드하기

    복제된 폴더 안에 html, js, css, 이미지 파일 등을 저장하고 업로드하면 그 내용이 그대로 서버에 반영되어 접속했을 때 보여지게 된다. 서버에 업로드하기 위한 절차가 2단계 있다.

    • commit
    • push to origin 이다.

    업로드할 파일 만들기

    복제된 폴더 안에(내 경우 d:\test\html5) docs 폴더를 만들자.

    d:\test\html5\docs 폴더가 생기는 거다.

    d:\test\html5\docs 폴더 안에 index.html 파일을 만든다. 그리고 index.html 파일에 내용으로 딱 한 줄만 적어보자.

    d:\test\html5\docs\index.html 파일의 내용:

    hi~

    내용 편집은 index.html 파일을 오른쪽 클릭해서 연결 프로그램 > 메모장 을 선택해 할 수 있다.

    커밋

    index.html 파일을 만들고 내용을 수정한 다음 다시 github desktop을 보면 다음과 같이 되어 있을 거다.

    왼쪽 윗 부분은 무언가 변경이 된 내용을 표시하고 있다. 사실 github는 단순히 파일을 업로드 하는 용도가 아니라 변화가 생길 때마다 버전 관리를 해 주는 프로그램이기 때문에 이런 변화가 생길 때마다 표시해 준다.

    [Commit 1 file to main] 버튼을 클릭한다.

    1개 파일을 메인에 커밋한다는 뜻의 버튼인데, 메인은 메인 저장소라는 뜻이고, 커밋은 우리가 작업한 내용(docs 폴더를 만들고, 그 안에 index.html을 생성, 수정한 것)을 하나의 묶음으로 저장하겠다는 뜻이다.

    개발자들이 사용하는 것이니 우리는 그냥 서버에 올리려면 [commit ~~~ ] 버튼을 눌러야 한다는 정도로만 기억하자.

    서버에 전송(push origin)

    커밋 버튼을 누르고 나면 github desktop이 이렇게 바뀐다. commit을 방금 헀기 때문에, 왼쪽 위처럼 현재는 바뀐 내용이 없다고 보여주고 있고, 방금 commit 한 내용을 서버에 전송하라는 버튼이 오른쪽에 나타난다.

    이 [push origin] 버튼을 누른다.

    서버에 잘 올라갔나 확인

    이제 다시 웹브라우저를 이용해 github 사이트에 내가 방금 만든 index.html 파일이 잘 올라갔나 확인해 보자.

    아마 웹브라우저가 좀 전에 로그인해서 위와 같은 상태일텐데, 그냥 주소창에 'github.com'을 입력해 github 사이트로 이동하자.

    왼쪽에 내가 만든 레포지토리 들이 나타나는데 그 중 html5 로 끝나는 레포지토리를 선택한다.

    그럼 좀전에 내가 만든 docs라는 폴더가 업로드 된 걸 볼 수 있다.

    클릭해서 그 안에 index.html 파일이 있는지도 확인해 보자.

    내가 만든 index.html 이 업로드되어 있는 걸 볼 수 있다.

    index.html 파일을 클릭해서 내용을 보자.

    내가 만든 내용 그대로 업로드된 걸 확인했다.

    내가 올린 내용으로 웹호스팅하겠다고 설정

    이제 내가 올린 내용으로 웹호스팅하겠다고 설정해야 한다.

    [Settings] 버튼을 누른다.

    Settings 페이지에서 [Pages] 메뉴를 선택하고,

    Branch 부분을 main, /docs 로 설정하고 [Save] 버튼을 누른다.

    [Save] 버튼을 눌렀으면 한 3분쯤 기다리자.

    /docs라고 설정해 놓으면 우리의 레포지토리에 /docs 폴더 안의 내용을 웹호스팅하겠다는 뜻이다. 다른 폴더에 저장한 내용은 웹호스팅에서 제외된다.

    웹사이트를 만들다 보면 웹서버에 접속했을 때 보여지는 것 외에도 많은 파일들을 생성하게 되는데(예를 들면 기획안, 디자인 시안, 개발일지 등등) 이런 것들은 웹호스팅을 통해 공개되지 않고 딱 정해진 내용만 공개할 수 있게 된다.

    3분쯤 후에 이 페이지를 [F5] 키를 눌러 refresh 해보면,

    윗부분이 이렇게 바뀐다. 이제 드디어 내가 올린 내용으로 웹호스팅을 받을 수 있게 된 거다.

    이제 전세계 어디에서도 24시간 웹호스팅을 해주는 주소가 생긴건데, 위 예에서는

    https://studentkim2000.github.io/html5/ 부분이 그 주소다.

    www.naver.com 처럼 나만의 웹서버 주소가 생겼다.

    웹브라우저에 저 주소를 다 타이핑해도 되지만,

    [Visit site] 버튼을 누르면 바로 이동한다.

    내 웹서버에 접속

    이렇게 내가 올린 내용이 그대로 반영되어 화면에 나타난다. 핸드폰으로도 접속 가능하다.

    폰으로도 접속

    핸드폰에서 내 주소를 다 타이핑해도 되지만 많이 불편하니 크롬 브라우저의 하얀색 빈 곳을 오른쪽 클릭하고 이 페이지의 QR 코드 생성 메뉴를 선택하자.

    이렇게 생긴 QR 코드를 핸드폰을 이용해 바로 내 사이트에 접속해 볼 수 있다.
