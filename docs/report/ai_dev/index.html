    소프트웨어 개발에 AI를 활용한 후기 - 2024

    2024년 한 해 동안 소프트웨어 개발에 AI를 활용해 본 개발자가 느낀 점을 솔직히 적어봄.

    개요

    • 사용한 AI: (구글) Gemini, (OpenAI) ChatGPT
    • 사용한 언어 및 framework: C/C++, rust, python, MS MAUI, MS WinUI 3, Godot 4
    • 작성일: 2024. 12. 30.(월)
    • 작성자: 김문식

    결론

    스스로 간단한 프로그램을 처음부터 끝까지 다 만드는 건 어느 정도 요구 사항을 비슷하게 충족시켜주고 있음.

    하지만 조금 규모가 있는 프로그램을 개발하면서 필요한 "부분 기능"들을 요청해 보면 아직 많이 부족함. 개발자들이 정말 원하는 것은 이런 것인데. 프로그램의 큰 틀을 만드는 걸 AI에게 맡기지는 않음.

    "부분 기능" 개발이 많이 부족하다고 했지만, python으로 개발할 때는 그래도 상황이 꽤 괜찮음. 실제 동작하는 코드를 보여주기도 함. python을 조금 할 줄 아는 수준만 되어도 큰 도움을 받을 수 있겠음. C, rust 등 다른 언어들은 한 번에 동작하는 코드를 받아본 적이 없음.

    가끔 거짓말 함. 해당 framework에서 전혀 사용할 수 없는 코드인데도 가능한 것처럼 코드를 만들어 보여줘서 가끔 놀라기도 함.

    활용 소감

    1. 될 듯 안 될 듯

    AI는 소프트웨어 개발 쪽 성과를 강조하기 위해서인지 스스로 프로그램을 만들어 내는 모습을 보여주는데 주력하고 있는 듯 하다. 이렇게 스스로 프로그램을 만드는 것은 대체로 간단한 프로그램이 경우가 대부분이고, 비슷하게는 만들어주지만 아직 내가 원한 복잡한 기능을 만들어 주지는 못하는 수준이다.

    예를 들어 코딩을 거의 전혀 못 하는 사람의 입장이 되어 "파이썬으로 구구단 보여주는 프로그램 만들어 줘" 라고 하면 이런 프로그램은 어떤 식으로든 만들어준다. 처음 보면 대단하다는 생각이 든다.

    하지만 개발자의 입장이 되어 프로그램 개발에 필요한 복잡하면서도 그 프로그램에서만 사용할만한 특유의 기능을 만들어달라고 하면 거의 제대로 만들어내질 못한다.

    쉬운 건 잘하는데, 특화되고 복잡한 건 아직 잘 못 한다.

    2. 오히려 시간이 더 많이 걸리기도

    스톱워치 프로그램을 만든다고 치자. 프로그램 내에서는 초(second) 단위로 모든 자료를 관리하지만 이걸 사용자에게 보여줄 때는 시:분:초의 형태로 바꿔 보여줘야 한다. 예를 들어 72초라고 사용자에게 보여주면 사람인 사용자는 굉장히 불편해 하기 때문에 이걸 1분 12초라고 바꿔 보여줘야 한다.

    이렇게 해주는 코드를 만들어 달라고 요청하면, 딱 원하는 것 보다는 대체로 비슷한 기능을 하는 코드를 만들어 준다. 예를 들면 "00:01:12 같은 식으로 보여지게 해 줘" 라고 요청해도 "0:1:12"와 같이 보여지는 코드를 만들어 주는 식이다.

    수정 요청을 해도 방금 전 제시한 것에서 결국엔 결과가 똑같거나 비슷한 결과가 나오는 코드를 만들어서 보여줬다.

    방법 자체를 완전히 달리 해 달라고 요청해도 역시나 거의 똑같은 결과가 나오는 코드를 만들어주는 경우가 대부분이었다.

    이 코드로는 안되겠는데 싶어서 새로운 방법으로 해달라고 요청한건데 결국엔 비슷비슷한 코드만 보여주는 경우가 많아 차라리 그냥 공식 문서를 찾아보거나 인터넷에서 사람이 쓴 글을 찾아보는 게 더 좋겠다는 생각이 들었다. 직접 검색하면 구글 검색 결과로 여러 글들의 목록이 보여지고 그 중 나에게 맞는 걸 찾아서 보면 되는데, AI는 자기가 만들어 낸 코드를 거의 바꾸지 않고 말만 조금 바꿔서 다시 보여주는 경우가 많아 결국엔 한 종류의 코드만 계속 보게되는 경우가 대부분이었다. 이건 상당한 문제이다. AI에게 요청하면 시간을 많이 들여도 사실상 비슷비슷한 코드만 여러 개 보는 결과가 된다.

    3. 어렵거나 최신 기술들은 더 잘 안 된다.

    현재 개발자들이 쓰는 언어 중 가장 어려운 언어는 아마 rust 일 거다. C/C++보다 진입장벽이 더 높다. 일단 C/C++을 알아야 이해할 수 있는 언어니까. AI는 rust에 대해 상당히 오래된 버전의 코드를 학습한 듯 하다. rust 언어로 개발 중 받은 코드는 제대로 동작하는 코드가 하나도 없었다. MS MAUI나 MS WinUI3 라는 비교적 최신 기술을 이용한 프로젝트의 경우에도 역시 제대로 동작하는 코드를 준 적이 없다. 게임엔진인 Godot 4 역시 마찬가지로 동작하는 코드를 받아보지 못했다.

    인터넷에 자료가 많아 학습이 용이한 경우(특히 파이썬)에는 동작하는 코드를 보여주지만 그렇지 않은 경우에는 동작하는 코드를 주지 못하는 경우가 대부분이었다.

    4. 가끔 거짓말

    AI가 잘못된 내용을 학습하면 잘못된 내용을 말할 수 밖에 없다. 코딩은 거짓말하기 어려울 거 같은데도 AI는 가끔 거짓말을 한다. 예를 들어 자동차에는 하늘을 나는 기능이 전혀 없는데, 자동차에게 하늘을 날라고 명령하는 코드를 만들어서 보여주는 식이다.

    개발 업계 전망

    앞으로 상황이 더 좋아질 것으로 기대한다. AI 자체의 성능이 매우 좋아지고 있으니까.

    개발팀이 규모있는 소프트웨어를 개발하는 흐름을 보면, 먼저 팀장이 설계를 하고 몇 개의 세션으로 나눈 후 각 세션을 개발할 개발자들이 어떤 능력을 갖춰야 하는지 정의한 후, 그 능력에 맞는 개발자들을 모아 개발 업무를 맡기고 각 세션별로 만들어진 코드를 모아 하나의 소프트웨어 제품으로 통합한다.

    앞으로 여기서의 각 세션 개발자들은 AI로 대체될 가능성이 높다. 당장 1, 2년 내로는 아니겠지만. 특히 신입 (실력 정도의) 개발자들은 더욱 더 설 자리가 없어질 것으로 보인다. 앞으로는 팀장급의 시니어 개발자 한 두명이 AI를 진두지휘하며 코드를 만들어가며 기존보다 높은 생산성으로 코드를 만들어 낼 것으로 보인다. 그래서 소프트웨어를 설계하고, 통합하고, AI가 만들어준 코드를 검증할 수 있고, 위험성을 파악해 수정할 수 있는 능력을 가진 시니어 개발자들만 살아남을 것으로 보인다.

    그리고 이러한 현상, 진짜 실력있는 사람, 특히 설계와 검증이 가능한 사람만 살아남아 큰 틀은 본인이 만들고(설계) 그 안에 필요한 업무들은 AI에게 시키며 일을 진행해 가는 방식은 앞으로 모든 분야에서 벌어질 것으로 생각된다. 그래서 본인 업무 분야에 대해 정말 잘 알고, AI를 잘 다루고, 필요하면 스스로 AI를 만들어 내는 능력을 가진 사람이 미래 사회에 가장 높은 위치에 서게 될 것이다.

    개발 교육에 관한 전망

    코딩 교육을 하면 AI가 다 만들어 주는데 이걸 왜 배워야 하느냐는 말을 하는 사람들이 있다고 들었다.

    앞으로 AI가 만들어 준 코드를 많이 사용하는 사회가 올 것이기 때문에 더더욱 코딩 교육을 해야 한다. 일반 업체에서는 주니어가 코드를 만들면 기본적으로 시니어가 리뷰해서 문제를 검증하고 문제가 없다는 판단이 서면 그 코드를 사용한다. 이런 절차없이, 더더군다나 인간이 최소한의 코드 검증조차 하지 않고 그냥 자료 몇 개 테스트해보고 되네 하며 그냥 사용하는 것은 큰 문제를 일으킨다. 예를 들어, 좌표 변환하는 코드를 AI가 만들어 줬는데 이걸 그대로 믿고 몇년 간 사용하다가 나중에 잘못이 밝혀지면 큰 재앙이 된다.

    AI가 만들어준 코드는 꼭 사람이 최소한의 검증은 해야 하기 때문에 사람도 어느 정도는 코딩 능력을 갖춰야 한다. 이 능력이 없으면 주변의 다른 사람들은 AI의 도움을 받아 스스로 필요한 프로그램을 만들어 자기 일을 쉽고 빠르게 처리하는데 나는 그러지 못해 점점 더 안 좋은 상황이 될 가능성이 상당하다. 앞으로 간단한 프로그램 정도는 AI의 도움으로 직접 만들어 쓰는 시대가 올 것이기 때문에 코딩 능력이 오히려 더 필요하다. 개발자가 대체되는 거지 일반 사람들은 지금보다 더 코딩을, 개발을 더 많이 하게 될 거다. 마치 자동차 운전이 더 쉬워지고 편해지니 대부분의 사람들이 직접 운전을 하는 것처럼.
